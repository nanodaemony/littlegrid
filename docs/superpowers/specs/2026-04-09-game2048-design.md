# 2048 游戏工具设计文档

## 1. 概述

### 1.1 目标
为 "小方格" Flutter 应用添加一个新的游戏工具：2048 数字合并游戏。

### 1.2 范围
- 基础 2048 游戏玩法

- 渐变色系 UI 设计

### 1.3 非目标

- 多人对战
- 后端 API 集成

## 2. 游戏机制

### 2.1 基础规则
- **棋盘**: 4×4 格子
- **初始**: 随机生成 2 个数字（90% 概率为 2，10% 概率为 4）
- **操作**: 上下左右滑动合并相邻相同数字
- **得分**: 每次合并相加（如合并 4+4=8，得 8 分）
- **新数字**: 每次有效移动后在空格随机生成新数字（2 或 4）

### 2.2 胜利与失败
- **胜利**: 出现 2048 数字时显示庆祝动画，但游戏可继续
- **失败**: 棋盘填满且无法进行任何合并

### 2.3 成就等级
| 等级 | 数字 | 徽章 |
|-----|------|-----|
| 入门 | 2048 | 🥉 |
| 进阶 | 8192 | 🥈 |
| 大师 | 32768 | 🥇 |
| 传奇 | 65536+ | 👑 |

## 4. 数据模型

### 4.1 Tile (格子)
```dart
class Tile {
  final int value;      // 数字值
  final int row;        // 行位置 (0-3)
  final int col;        // 列位置 (0-3)
  final bool isNew;     // 是否新生成（用于动画）
  final bool isMerged;  // 是否刚合并（用于动画）
}
```

### 4.2 GameState (游戏状态)
```dart
class GameState {
  final List<Tile> tiles;           // 所有格子
  final int score;                  // 当前得分
  final int bestScore;              // 历史最高分
  final int maxTile;                // 当前最大数字
  final bool isGameOver;            // 是否结束
  final bool isWon;                 // 是否达到 2048
  final List<List<int>>? history;   // 撤销历史（最多3步）
  final DateTime startTime;         // 游戏开始时间
}
```

### 4.3 GameStats (游戏统计)
```dart
class GameStats {
  final int totalGames;      // 总游戏次数
  final int winCount;        // 达到 2048 的次数
  final int highestScore;    // 历史最高分
  final int maxTileReached;  // 达到过的最大数字
  final int totalPlayTime;   // 总游戏时长(秒)
}
```

## 5. 核心逻辑

### 5.1 移动算法
1. 压缩: 将非零数字向移动方向靠拢
2. 合并: 相邻相同数字相加合并
3. 再压缩: 合并后再次靠拢
4. 生成: 在空格随机生成新数字(2 或 4)
5. 检查: 判断游戏是否结束

### 5.2 撤销机制
- 保存最近 3 步的游戏状态
- 撤销时恢复到上一步状态
- 每次有效移动后更新历史

### 5.3 持久化
使用 `shared_preferences` 保存:
- 历史最高分
- 达到的最大数字
- 游戏统计
- 未完成的游戏状态（可选）

## 10. 风险与缓解

| 风险 | 影响 | 缓解措施 |
|-----|-----|---------|
| 大数字颜色重复 | 低 | 定义足够多的颜色，大数字使用计算渐变色 |
| 动画性能问题 | 中 | 使用 RepaintBoundary，限制同时动画数量 |
| 手势冲突 | 低 | 棋盘区域使用 GestureDetector 独占手势 |
